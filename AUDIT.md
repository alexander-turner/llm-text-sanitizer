# Security / robustness / testing / packaging audit — `agent-input-sanitizer`

Date: 2026-06-28. Every finding below was **confirmed with a runnable proof-of-concept**
(not static reading) — the Read tool silently strips control/zero-width/ANSI bytes, so all
byte-level claims were checked via `cat -v`/`od -c` and live `node`/`python` runs.

Severity: **CRIT** breaks all consumers · **HIGH** defeats a core defense or violates a
documented contract · **MED** real but bounded · **LOW** hygiene/UX.

### Security — defense defeats

| #   | Sev      | Finding                                                                                                                                                                                                                                                                                                                    | Evidence (PoC)                                                                                                                           |
| --- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| S1  | **HIGH** | `classifyPrompt` `hasAnsi` gate is `/\x1b/` (7-bit ESC only). A prompt with a pure **8-bit C1** CSI (U+009B) or OSC (U+009D) erase/cursor/title sequence and no other invisibles returns `{action:"pass"}` while its 7-bit twin **blocks**. `stripAnsiFully` removes the C1 seq, so `invisibleCount→0` and it reads clean. | `prompt.mjs:31,80`. `classifyPrompt(C1+"2J")`→`pass`; `ESC+"[2J"`→`block`. `isSgrOnly`/Layer-1 _do_ handle C1 — only this gate is blind. |
| S2  | **HIGH** | ZWNJ/ZWJ linguistic carve-out is an intact zero-width covert channel: the joiner cap limits only _consecutive_ joiners (resets at any 2-visible-char gap), so `letter joiner letter letter …` keeps every joiner under the scatter floor (30) — **silently**, `found:[]`.                                                  | `invisible.mjs:246-290`. 25 joiners between Arabic letters → 25/25 preserved, `found:[]`; full bitstream recoverable.                    |
| S3  | MED      | Layer-2 hidden-CSS misses two **unambiguously invisible** cases: `transform:scale(1e-3)` / `translateX(-1e4px)` (regex stops at `e`, reads `scale(1)`) and **negative** `opacity:-1` (`Math.abs` flips it to visible). Hidden content reaches the model.                                                                   | `html.mjs:99-111,225`. `isHiddenStyle("transform:scale(1e-3)")`→false; SECRET in such a span survives `sanitizeHtml`.                    |
| S4  | MED      | Layer-3 exfil blob detectors only allow standard base64 `[A-Za-z0-9+/]`; **base64url** (`-`/`_`, the natural in-URL encoding) escapes the query, fragment, and path-beacon detectors.                                                                                                                                      | `html.mjs:795,805`. `checkExfilUrl(".../c?data="+b64url)`→null; standard-alphabet twin is caught.                                        |
| S5  | MED      | `sanitizeValue` recurses values only; **object keys** are copied verbatim. A tool-output key carrying a zero-width run + ANSI passes through `modified:false`, no warning.                                                                                                                                                 | `output.mjs:320-332`. Confirmed: key retains ZWSP+ESC, zero warnings.                                                                    |

### Parallelism / sequencing notes

- **Fully independent (run now, any order):** PR-3, PR-4, PR-5, PR-6, PR-7, PR-8, PR-9, PR-11.
  File sets are disjoint; PR-3 fixes `prompt.mjs` with a _local_ introducer regex so it does **not**
  depend on PR-4's `invisible.mjs` changes.
- **PR-1 → PR-2 ordering (soft):** PR-2's new pack-&-install smoke job stays red until PR-1's
  `postinstall` fix lands. Land PR-1 first, or include the `postinstall` guard in PR-2's first commit.
- **README single-owner:** only PR-12 edits `README.md` (P4's "client is checkout-only / now
  installable" note and P9's `./view-map` row both go there) so PR-10 and PR-12 don't collide.
- **`tests/test_python_client.py` single-owner:** PR-10 owns it (its golden-corpus assertion from
  T9 lives there); PR-2 only edits workflows, PR-11 only edits JS test files.
- **Shared SSOT (`THREAT_CODEPOINTS`)** introduced by PR-11 is consumed by PR-3/PR-4/PR-5 property
  suites; if PRs land before PR-11, each defines its alphabet inline and PR-11 refactors to the SSOT.

### Suggested fix sketches (one line each)

- **P1:** drop `postinstall`; wire hooks via a `prepare` script guarded by `git rev-parse --is-inside-work-tree`.
- **P2:** add `*.egg-info/` to `.gitignore`; narrow `files` to `src/*.mjs`,`bin/sanitize-cli.mjs`; CI greps the tarball for `egg-info`.
- **S1:** `const ANSI_INTRODUCER = /[]/;` gate `hasAnsi` on it (reuse Layer-1's introducer set as SSOT).
- **S2:** cap **total** preserved-via-carve-out joiners per string (small budget), and flag (`found`) whenever any joiner is preserved past it.
- **S3:** extend transform numeric capture to `[-+]?\d*\.?\d+(?:[eE][-+]?\d+)?` + `parseFloat`; treat `opacity < ε` as hidden (drop `Math.abs`).
- **S4:** add a base64url arm (`[A-Za-z0-9_-]{40,}={0,2}`) to the query/fragment/path blob detectors.
- **S5:** sanitize keys; since a changed key alters shape, **flag/deny** rather than silently rewrite.
- **R1:** wrap the rehype/remark parse in `sanitizeHtml`/`detectExfil` in try/catch → fail **closed** (placeholder + warning), so the boundary never throws.
- **R2:** in worker mode read raw chunks and abort/emit the error once the current unterminated line exceeds `maxInputBytes()`.
- **R3:** add a depth cap (suppress past it) + `WeakSet` cycle guard in `sanitizeValue` and `suppressToolOutput`.
- **R4:** `occurrences` returns `[]` for an empty needle (or throws a typed error); guard `old_string===""` in `rehydrateEdit`; fix the false comment.
- **R5:** create the temp with an unpredictable name and `flag:"wx"` (exclusive, `O_EXCL`-style) so an existing symlink fails loud.
